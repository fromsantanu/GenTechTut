# Chapter 19: Sampling Process (Why Generation Is Slow)

---

# 📘 Introduction

In previous chapters, we learned:

* Forward diffusion adds noise
* Reverse diffusion removes noise
* Diffusion models learn by predicting noise
* Training is stable because optimization is smooth

Now we study:

# 🌫️ The Sampling Process

This is the stage where the trained diffusion model actually generates images.

It is also where one major limitation appears:

> Diffusion generation is usually slow.

---

# 🧠 What is Sampling?

Sampling means:

> Generating a new image from random noise using the learned reverse diffusion process.

---

# 🎯 Basic Idea

The process starts with:

```id="k7hm5e"
Pure Random Noise
```

Then the model:

* Predicts noise
* Removes noise
* Repeats many times

Eventually:

* A realistic image emerges

---

# 📘 Reverse Sampling Flow

```id="8z1c3j"
Noise x_T
    ↓
x_(T-1)
    ↓
x_(T-2)
    ↓
...
    ↓
x_0
```

Where:

* ( x_T ) = pure noise
* ( x_0 ) = final generated image

---

# 🧠 Why Sampling Requires Many Steps

Diffusion models generate images gradually.

At each timestep:

* Only a small amount of noise is removed

This creates:

* High-quality generation
* But slow inference

---

# 🧩 Simple Analogy

Imagine sculpting a statue from stone.

You do not:

* Instantly create the final sculpture

Instead:

* Small refinements happen repeatedly

Diffusion generation works similarly.

---

# ⚙️ Step-by-Step Sampling Process

---

# Step 1️⃣ Start with Random Noise

The process begins with:

$$x_T \sim \mathcal{N}(0,I)$$

Meaning:

* Random Gaussian noise

---

# Step 2️⃣ Predict Noise

The neural network receives:

* Current noisy image
* Current timestep

It predicts:

* Noise component

---

# Step 3️⃣ Remove Predicted Noise

Using the predicted noise:

* The image becomes slightly cleaner

---

# Step 4️⃣ Repeat Again

The process continues:

* Many times

Until:

* Noise disappears
* Structure emerges

---

Initially:

---

# ⚠️ Why Diffusion Sampling Is Slow

This is one of the biggest limitations of diffusion models.

---

# 📌 Reason 1: Many Iterative Steps

Generation may require:

* 20 steps
* 50 steps
* 100 steps
* Sometimes 1000+ steps

Each step requires:

* Neural network computation

---

# 📌 Reason 2: Large Neural Networks

Modern diffusion systems use:

* U-Net architectures
* Attention layers
* Transformer components

These are computationally expensive.

---

# 📌 Reason 3: High Resolution Processing

Generating detailed images requires:

* Large tensors
* High memory usage
* Intensive GPU computation

---

# ⚖️ GAN vs Diffusion Speed

| Aspect           | GAN               | Diffusion            |
| ---------------- | ----------------- | -------------------- |
| Generation style | One forward pass  | Many denoising steps |
| Speed            | Very fast         | Slower               |
| Stability        | Difficult         | Stable               |
| Diversity        | Sometimes limited | High                 |

---

# 🧠 Why GANs Are Faster

GAN generators directly map:

```id="vq5t4n"
Noise → Image
```

in one pass.

Diffusion models instead perform:

```id="evm8np"
Noise → Slightly Cleaner Noise
→ Cleaner Noise
→ Image
```

---

# 📘 DDPM (Denoising Diffusion Probabilistic Model)

One important diffusion approach is:

# 📘 DDPM

---

# 🧠 Basic Idea

DDPM uses:

* Many small denoising steps
* Probabilistic reverse transitions

---

# 📌 Advantage

Produces:

* High-quality images
* Stable generation

---

# 📌 Limitation

Sampling can be slow.

---

# 📘 DDPM Reverse Equation

p_\theta(x_{t-1}|x_t)

---

# 🧠 Meaning

The model predicts:

* Cleaner image distribution at each step

---

# ⚡ Faster Sampling: DDIM

Researchers developed faster methods such as:

# 📘 DDIM (Denoising Diffusion Implicit Models)

---

# 🧠 Goal

Reduce:

* Number of denoising steps

While preserving:

* Image quality

---

# ⚖️ DDPM vs DDIM

| Feature       | DDPM          | DDIM               |
| ------------- | ------------- | ------------------ |
| Sampling      | Probabilistic | More deterministic |
| Speed         | Slower        | Faster             |
| Image Quality | Very high     | High               |

---

# 🧠 Tradeoff Between Speed and Quality

This is an important concept.

---

# 📉 Fewer Steps

Advantages:

* Faster generation

Disadvantages:

* Lower detail
* Possible artifacts

---

# 📈 More Steps

Advantages:

* Better realism
* Better refinement

Disadvantages:

* Slower inference

---

# 🧠 Sampling Schedulers

Modern systems use:

* Specialized schedulers

These control:

* How denoising progresses

---

# 📘 Common Schedulers

| Scheduler        | Purpose              |
| ---------------- | -------------------- |
| Linear scheduler | Simple progression   |
| Cosine scheduler | Smooth denoising     |
| Euler scheduler  | Faster approximation |
| LMS scheduler    | Stable sampling      |

---

# ⚙️ Latent Diffusion Models

Modern systems such as:

Stable Diffusion

use:

# 📘 Latent Diffusion

---

# 🧠 Important Idea

Instead of denoising full images:

* The model denoises compressed latent representations

This greatly reduces:

* Computation
* Memory usage

---

# 🧩 Simple Analogy

Instead of editing:

* A full high-resolution image

You edit:

* A compressed sketch representation

Then reconstruct the final image.

---

# 🧠 Why Latent Diffusion Became Important

Latent diffusion enabled:

* Faster generation
* Consumer GPU usage
* Practical text-to-image systems

---

# 📘 Text-Guided Sampling

Modern diffusion systems combine:

* Text encoders
* Diffusion denoising

---

# 📌 Workflow

```id="j0vjlwm"
Text Prompt
      ↓
Text Embedding
      ↓
Guided Denoising
      ↓
Generated Image
```

---

# 🧠 Classifier-Free Guidance

Modern systems often use:

# 📘 Classifier-Free Guidance (CFG)

---

# 📌 Purpose

Improve:

* Prompt alignment

---

# 🧠 Simple Idea

The model is guided toward:

* Stronger matching with the text prompt

# ⚠️ Sampling Challenges

Even advanced diffusion systems still face:

---

# ❌ Slow Inference

Large models require:

* Significant GPU power

---

# ❌ Memory Usage

High-resolution generation is expensive.

---

# ❌ Long Rendering Time

Detailed images may require:

* Many denoising steps

---

# 🧠 Why Diffusion Is Still Dominant

Despite slower generation, diffusion models became dominant because they provide:

* Stable optimization
* High diversity
* Excellent realism
* Strong controllability

---

# ⚖️ Stability vs Speed Tradeoff

This is one of the major design tradeoffs in generative AI.

| Goal                           | Best Approach |
| ------------------------------ | ------------- |
| Faster generation              | GAN           |
| Stable high-quality generation | Diffusion     |

---

# 🧭 Big Picture Understanding

The diffusion sampling process is essentially:

> Gradually transforming randomness into structure through repeated refinement.

Generation quality comes from:

* Many small denoising improvements

But those repeated improvements create:

* Computational cost
* Slow generation speed

---

# 🎯 Key Insight

Diffusion generation is slow because:

> The model builds images progressively through many denoising iterations instead of generating everything in one step.

This gradual refinement is also what gives diffusion models:

* High realism
* Stability
* Strong diversity
